AIFIX – 공급망 관리 탭 구조 통합 설계

너는 제조업 공급망 ESG/PCF 관리 SaaS AIFIX의 시니어 프로덕트 디자이너다.

현재 협력사 포털에서 프로젝트를 선택하면 협력사 차수(1차/2차/3차)에 따라 “공급망 관리” 탭 UI가 서로 다르게 구성되어 있다.

그러나 실제 시스템 로직상 모든 협력사는 자신의 직하위 협력사를 관리할 수 있다.

예시:

원청
→ 1차 협력사
→ 2차 협력사
→ 3차 협력사

각 협력사는 다음 권한을 가진다.

자신의 직하위 협력사 등록

직하위 협력사에게 데이터 요청

직하위 협력사의 제출 상태 확인

따라서 협력사 차수에 따라 UI를 다르게 만들지 말고, 동일한 “공급망 관리 화면”을 사용하고 권한만 다르게 동작하도록 UI를 재설계하라.

1️⃣ 화면 목표 (Goal)

이 화면은 다음을 위한 공급망 관리 허브다.

사용자는 여기에서

자신의 하위 공급망 구조를 확인하고

협력사를 관리하고

데이터 요청 상태를 확인할 수 있어야 한다.

즉 이 화면은 단순 조회가 아니라

공급망 관리 + 협력사 관리 + 데이터 수집 상태 관리 화면이다.

2️⃣ UI 구조 (3 Panel Layout)

화면은 3개의 영역으로 구성한다

좌측 → 공급망 트리
중앙 → 협력사 목록
우측 → 협력사 상세

3️⃣ 좌측 영역 — 공급망 트리 (Supply Chain Tree)

좌측 패널에는 공급망 트리 구조를 표시한다.

구조 예시

6️⃣ 권한 제어 (RBAC)

UI는 동일하지만 권한에 따라 버튼 노출을 제어한다.

권한 규칙

사용자	하위 협력사 초대	데이터 요청
원청	가능	가능
1차	가능	가능
2차	가능	가능
3차	불가능	불가능

3차는 조회 전용 UI가 된다.

단 UI 구조는 동일하게 유지한다.

7️⃣ 디자인 원칙

이 화면은 다음 SaaS 디자인 패턴을 따른다.

SRM 시스템 스타일 공급망 관리 UI

좌측 트리 + 중앙 목록 + 우측 상세 구조

상태 기반 관리 UI

협력사 중심 관리 구조

9️⃣ UX 목표

사용자는 자신의 차수와 관계없이

공급망 구조를 직관적으로 이해하고

직하위 협력사를 관리하고

데이터 제출 상태를 확인할 수 있어야 한다.

이 화면은 협력사 포털에서 가장 핵심적인 공급망 관리 화면이다.
