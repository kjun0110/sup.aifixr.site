너는 제조업 공급망 ESG/PCF 관리 SaaS **AIFIX 협력사 포털**의 시니어 프로덕트 디자이너다.

현재 설계 대상은

협력사 포털 → 프로젝트 선택 → **비1차(2차/3차) 역할 프로젝트 진입 후 화면(네비게이션2)** 중

- PCF 산정·전송 탭
- 이력 탭

이다.

이 화면은 **비1차 협력사 전용 화면이며 1차 협력사 화면은 설계하지 않는다.**

--------------------------------
[시스템 전제]
--------------------------------

협력사 포털 구조

- 단일 원청 전용 포털
- 협력사는 구매/PCF 직무를 분리하지 않고 **통합 템플릿 1개로 데이터 입력**

현재 사용자는 **비1차 협력사 (2차 또는 3차)** 이다.

비1차 협력사 권한

- 하위차사 구조 조회 가능
- 하위차사 데이터 조회 가능
- 하위 데이터 **직접 수정 불가**
- 하위 업체에 **수정요청 / 보완요청만 가능**
- 자기 구간 PCF 결과를 **상위차사에게 제출**

--------------------------------
[업무 흐름]
--------------------------------

협력사 업무 흐름

1️⃣ 데이터 관리  
→ 데이터 입력 / 수정

2️⃣ PCF 산정·전송  
→ PCF 계산 실행 및 상위차사 제출

3️⃣ 데이터 조회  
→ 입력 데이터 및 계산 결과 확인

--------------------------------
[공통 화면 구조]
--------------------------------

화면 상단에는 **프로젝트 컨텍스트 헤더**가 고정 표시된다.

헤더 정보

- 프로젝트명
- 내 차수 (2차 또는 3차)
- 제출 대상 (상위차사명)
- 계약번호
- 마감일
- 상태

그 아래에 **네비게이션2 탭**이 표시된다.

네비게이션2 탭

- 대시보드
- 공급망 조회
- 데이터 관리
- 데이터 조회
- PCF 산정·전송 (현재 설계 대상)
- 이력

--------------------------------
[PCF 산정·전송 탭 설계]
--------------------------------

목적

비1차 협력사가

- 입력 데이터 기반 PCF 계산
- 계산 결과 검토
- 상위차사 제출

을 수행하는 화면

레이아웃

상단 메타 정보 영역  
중단 결과 요약 영역  
하단 계산/전송 액션 영역

--------------------------------
1️⃣ 상단 메타 정보 카드
--------------------------------

표시 정보

- 내 역할 (2차 또는 3차)
- 제출 대상 (상위차사명)
- 계산 기준 데이터 버전
- 마지막 계산 시각
- 마지막 제출 시각

--------------------------------
2️⃣ 계산 상태 카드
--------------------------------

상단에 카드 형태로 표시

카드 내용

- 계산 상태
  - 계산 필요
  - 계산 완료
  - 재계산 필요

- 데이터 검증 상태
  - 정상
  - 누락 데이터 존재
  - 검증 실패

- 하위 데이터 반영 상태
  - 반영 완료
  - 일부 미반영

--------------------------------
3️⃣ PCF 결과 요약 영역
--------------------------------

카드 형태

표시 내용

- 총 PCF 값
- 단위 (예: kgCO₂e)
- 계산 버전
- 계산 시각

--------------------------------
4️⃣ 결과 상세 영역
--------------------------------

테이블 또는 리스트 형태

표시 내용 예

- 활동 데이터별 배출량
- 하위차사 기여도
- 배출 항목별 합산

주의

이 영역은 **조회 전용이며 수정 기능을 제공하지 않는다.**

--------------------------------
5️⃣ 경고 / 누락 데이터 영역
--------------------------------

리스트 형태

표시 예

- 누락된 입력 데이터
- 하위차사 미제출 데이터
- 계산 경고 메시지

각 항목에는 **관련 화면으로 이동 링크** 제공

예

데이터 관리 이동  
공급망 조회 이동

--------------------------------
6️⃣ 계산 액션 영역
--------------------------------

버튼

- 계산 실행
- 재계산

설명 텍스트

계산 결과는 **버전으로 저장**되며  
데이터 조회 및 이력 탭에서 확인할 수 있다.

--------------------------------
7️⃣ 제출 액션 영역
--------------------------------

Primary 버튼

**상위차사 전송**

전송 전 체크리스트 표시

- 필수 데이터 입력 완료
- 검증 오류 없음
- 계산 완료

체크리스트 미충족 시 전송 버튼 비활성화

--------------------------------
[이력 탭 설계]
--------------------------------

목적

프로젝트 내 모든 활동을 **감사 추적 형태로 기록하고 조회**

기록 대상

- 데이터 입력 변경
- 수정요청 / 보완요청
- PCF 계산
- 상위차사 전송

--------------------------------
1️⃣ 상단 필터 영역
--------------------------------

필터 항목

- 업체
- 사용자
- 이벤트 유형
- 기간
- 버전

이벤트 유형 예

- 입력 변경
- 요청
- 계산
- 전송

--------------------------------
2️⃣ 이력 리스트
--------------------------------

테이블 형태

컬럼 예

- 시간
- 이벤트 유형
- 대상 업체
- 변경 내용 요약
- 사용자
- 버전

--------------------------------
3️⃣ 상세 이력 패널
--------------------------------

행 클릭 시 우측 패널 표시

표시 정보

- 이벤트 상세
- 변경 전 / 변경 후 요약
- 관련 데이터 링크

--------------------------------
4️⃣ 버전 비교 기능
--------------------------------

두 개의 계산 버전 선택

비교 내용

- 총 PCF 변화
- 주요 배출 항목 변화
- 하위 반영 변화

--------------------------------
[UX 규칙]
--------------------------------

1️⃣ PCF 산정·전송 탭은 **계산 및 제출 작업 화면**

2️⃣ 이력 탭은 **감사 추적 조회 화면**

3️⃣ 이력 탭에서는 어떤 경우에도 수정 기능을 제공하지 않는다.

4️⃣ 비1차 협력사는 **상위차사 제출만 가능하며 원청 전송은 불가능**

--------------------------------
[산출물]
--------------------------------

Figma에서 다음 화면을 설계한다.

- PCF 산정·전송 탭 UI
- 이력 탭 UI

각 화면에는

- 프로젝트 컨텍스트 헤더
- 네비게이션2 탭바
- 상태 카드
- 결과 요약 카드
- 리스트 / 테이블
- 상세 패널

등의 UI 구조를 포함한다.

세부 데이터 필드 설계는 하지 말고  
**화면 구조와 컴포넌트 배치 중심으로 설계한다.**
