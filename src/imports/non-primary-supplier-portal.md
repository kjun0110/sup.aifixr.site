너는 제조업 공급망 ESG/PCF 관리 SaaS **AIFIX 협력사 포털**의 시니어 프로덕트 디자이너다.

현재 설계 대상은  
협력사 포털 → 프로젝트 선택 → **비1차(2차/3차) 역할 프로젝트 진입 후 화면(네비게이션2)** 이다.

이번 작업에서는 네비게이션2 중 아래 두 개 탭만 설계한다.

- 대시보드
- 공급망 조회

이 화면은 **비1차 협력사 전용 화면이며 1차 화면은 설계하지 않는다.**

--------------------------------
[시스템 고정 전제]
--------------------------------

협력사 포털은 다음 구조를 가진다.

- 단일 원청 전용 포털
- 협력사는 구매/PCF 직무를 분리하지 않고 하나의 통합 템플릿으로 데이터 입력

현재 사용자는 **비1차 협력사 (2차 또는 3차)** 이다.

비1차 협력사 권한

- 하위차사 구조 조회 가능
- 하위차사 데이터 조회 가능
- 하위 데이터 직접 수정 불가
- 하위 업체에 수정요청 / 보완요청만 가능
- 자기 구간 PCF 결과를 **상위차사에게 제출**

--------------------------------
[공통 레이아웃]
--------------------------------

프로젝트 진입 후 화면 상단에 **프로젝트 컨텍스트 헤더**를 표시한다.

헤더 정보

- 프로젝트명
- 내 차수 (2차 또는 3차)
- 제출 대상 (상위차사명)
- 계약번호
- 마감일
- 상태

그 아래에 **네비게이션2 탭**이 표시된다.

네비게이션2 탭

- 대시보드
- 공급망 조회
- 데이터 관리
- 데이터 조회
- PCF 산정·전송
- 이력

이번 설계 범위는 **대시보드와 공급망 조회 탭만**이다.

--------------------------------
[대시보드 탭 설계]
--------------------------------

대시보드는 **비1차 협력사의 업무 시작 화면**이다.

목적

- 현재 데이터 입력 상태 확인
- 하위 업체 제출 상태 확인
- 수정요청 진행 상황 확인
- 상위차사 제출 준비 상태 확인

레이아웃은 **상단 KPI 카드 + 하단 운영 리스트 구조**로 설계한다.

--------------------------------
1️⃣ 상단 KPI 카드 영역
--------------------------------

최소 4개의 상태 카드 배치

카드 내용

1. 자사 데이터 입력 진행률
- 입력 완료율 %

2. 하위차사 제출 현황
- 제출 완료 업체 수 / 전체 업체 수

3. 수정요청 진행
- 요청 중 / 처리 완료

4. 상위차사 제출 상태
- 미제출
- 제출 대기
- 제출 완료

각 카드에는 **빠른 이동 링크**를 제공한다.

예

자사 데이터 입력 → 데이터 관리 이동  
하위 제출 현황 → 공급망 조회 이동  
상위 제출 상태 → PCF 산정·전송 이동

--------------------------------
2️⃣ 하단 "하위차사 제출 현황 리스트"
--------------------------------

테이블 또는 카드 리스트 형태

컬럼 예

- 업체명
- 차수
- 납품 항목
- 제출 상태
- 최근 제출일
- 요청 상태

행 액션

- 상세 보기
- 수정요청
- 보완요청

주의

이 화면에서는 **직접 수정 버튼을 절대 표시하지 않는다.**

--------------------------------
3️⃣ 알림/경고 영역
--------------------------------

대시보드 하단 또는 우측 사이드에 배치

알림 예

- 마감 임박 프로젝트
- 미제출 하위 업체
- 수정요청 응답 없음

각 알림에는 **바로 이동 링크**를 제공한다.

--------------------------------
[공급망 조회 탭 설계]
--------------------------------

이 탭은 **공급망 구조를 확인하고 하위 업체 상태를 관리하는 화면**이다.

비1차 협력사는 공급망을 조회할 수 있지만  
**하위 데이터 직접 수정 권한은 없다.**

레이아웃

좌측

공급망 트리

우측

업체 상세 패널

--------------------------------
1️⃣ 좌측 영역: 공급망 트리
--------------------------------

트리 구조로 공급망 표시

표시 정보

- 업체명
- 차수
- 제출 상태
- 최근 제출일

트리 레벨 예

상위차사  
→ 현재 사용자  
→ 하위차사

현재 사용자는 트리에서 강조 표시한다.

--------------------------------
2️⃣ 공급망 필터 영역
--------------------------------

트리 상단에 필터 제공

필터 항목

- 차수
- 제출 상태
- 납품 항목
- 기간

검색 기능

- 업체명 검색

--------------------------------
3️⃣ 우측 영역: 업체 상세 패널
--------------------------------

트리에서 업체 선택 시 상세 패널 표시

패널 구성

섹션1

업체 기본 정보

- 업체명
- 차수
- 납품 항목
- 계약번호

섹션2

데이터 제출 상태

- 제출 여부
- 최근 제출일
- 데이터 버전

섹션3

요청 이력

- 수정요청
- 보완요청
- 최근 요청 날짜

--------------------------------
4️⃣ 허용 액션 (비1차 전용)
--------------------------------

우측 패널에 아래 버튼 제공

- 수정요청
- 보완요청

이 버튼을 누르면 요청 메시지를 입력하는 모달을 띄운다.

중요

**직접 수정 버튼은 절대 제공하지 않는다.**

--------------------------------
[UX 규칙]
--------------------------------

비1차 공급망 조회 탭은 다음 원칙을 따른다.

1. 공급망 구조는 조회 중심 UI로 설계한다.
2. 데이터 직접 수정 기능은 존재하지 않는다.
3. 하위 업체에 대한 조치는 “수정요청 / 보완요청”으로만 처리한다.
4. 공급망 관리 권한은 1차에 있으므로 관리 기능은 최소화한다.

--------------------------------
[산출물]
--------------------------------

Figma에서 다음 화면을 설계한다.

- 비1차 협력사 프로젝트 대시보드 화면
- 비1차 협력사 공급망 조회 화면

각 화면에는

- 상단 프로젝트 컨텍스트 헤더
- 네비게이션2 탭바
- KPI 카드
- 상태 리스트
- 공급망 트리
- 업체 상세 패널

등의 UI 구조를 포함한다.

세부 데이터 필드 설계는 하지 말고  
**화면 구조와 컴포넌트 배치 중심으로 설계한다.**
