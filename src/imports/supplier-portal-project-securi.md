AIFIX 협력사 포털 – 프로젝트 선택 화면 보안 구조 수정

너는 제조업 공급망 ESG/PCF 관리 SaaS AIFIX의 시니어 프로덕트 디자이너다.

현재 “협력사 포털 – 프로젝트 선택” 화면은
협력사가 전체 공급망 내 자신의 차수(Tier)를 추측할 수 있는 UI 구조를 가지고 있다.

공급망 보안 정책상 협력사는 자신이 몇 차 협력사인지 알 수 없어야 한다.

따라서 다음 기준에 따라 화면을 수정하라.

1️⃣ 차수(Tier) 정보 완전 제거

현재 프로젝트 카드 좌측 상단에 표시된

1차

2차

3차

와 같은 차수 배지(Tag)를 완전히 제거한다.

또한 아래 요소도 함께 삭제한다.

삭제 대상

상단 필터

전체 역할
  - 1차 협력사
  - 2차 협력사
  - 3차 협력사

→ 전체 역할 드롭다운 필터 자체를 삭제

협력사는 프로젝트의 차수 개념을 UI 어디에서도 볼 수 없어야 한다.

2️⃣ 프로젝트 카드 정보 구조 수정

현재 카드에는 다음 정보가 보인다.

상위차사
납품 항목
계약번호
마감일
최근 업데이트

하지만 상위차사 정보는 간접적으로 공급망 구조를 추론하게 할 수 있다.

따라서 다음 방식으로 수정한다.

기존
상위차사 : 원청사 A
변경
요청 기업 : 원청사 A

또는

요청 기업 : ○○전자

상위차사 / 하위차사 개념은 UI에서 제거한다.

협력사는 단지

“이 기업이 나에게 데이터를 요청했다”

라고만 인지해야 한다.

5️⃣ 프로젝트 예시 데이터 수정

현재 6개의 프로젝트가 보이는데
이 프로젝트명은 차수 기반이 아닌 공급망 역할 기반 컨셉으로 수정한다.

단 차수(1차/2차/3차)는 절대 표기하지 않는다.

상단 3개 프로젝트 (배터리 공급망 컨셉)

배터리 산업 공급망 단계를 반영하되
차수 표현 없이 작성한다.
