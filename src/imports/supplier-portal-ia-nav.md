너는 제조업 공급망 ESG/PCF 관리 SaaS **AIFIX**의 시니어 프로덕트 디자이너다.
이번 작업의 목적은 **협력사 포털의 정보구조(IA)와 네비게이션 구조를 설계하되, AIFIX 브랜드의 공통 디자인 톤까지 일관되게 반영하는 것**이다.

중요:
이번 단계에서는 **세부 탭 내부 콘텐츠까지 설계하지 말고**,
오직 아래 범위까지만 설계하라.

* **로그인 후 보이는 협력사 포털 메인 레이아웃**
* 그 메인 레이아웃 안의 **전역 네비게이션1**
* 로그인 직후 기본으로 열리는 **네비게이션1의 프로젝트 탭 화면(= 프로젝트 선택 화면)**
* 프로젝트 선택 후, 해당 프로젝트가 **1차 / 2차 / 3차일 때 각각 보이는 네비게이션2 구조**
* 각 네비게이션 탭 이름, 계층, 화면 전환 관계, 레이아웃 골격

**세부 탭 안에 들어갈 표/폼/카드/데이터 상세내용은 아직 만들지 말 것.**
지금은 어디까지나 **구조 설계 단계**다.

---

# 0. 협력사 포털의 전제

이 포털은 **하나의 원청사 전용 협력사 포털**이다.
즉, 여러 원청이 섞인 멀티테넌트 포털이 아니라,
**특정 원청과 연결된 협력사들이 사용하는 단일 포털**이다.

한 협력사는 이 포털 안에서 여러 프로젝트에 참여할 수 있다.
그리고 프로젝트마다 그 협력사의 역할은 달라질 수 있다.

예를 들어 같은 협력사라도:

* 어떤 프로젝트에서는 **1차**
* 어떤 프로젝트에서는 **2차**
* 어떤 프로젝트에서는 **3차**

가 될 수 있다.

따라서 협력사 포털은
**회사 단위의 전역 영역**과
**프로젝트 단위의 업무 영역**을 분리해서 설계해야 한다.

---

# 1. 화면 구조의 기본 원칙

협력사 포털은 로그인 후 아래 2단 구조를 가져야 한다.

## 네비게이션1 = 로그인 후부터 보이는 전역 네비게이션

협력사가 로그인해서 포털에 진입하면 보이는 **전역 메뉴**다.
프로젝트 선택 전에도 존재하며, 프로젝트에 들어간 후에도 유지된다.
회사 차원의 공통 관리 메뉴만 포함한다.

## 네비게이션2 = 프로젝트 선택 후 보이는 프로젝트 네비게이션

사용자가 네비게이션1의 **프로젝트 탭**에서 특정 프로젝트를 선택한 이후에만 나타나는
**프로젝트 전용 내부 네비게이션**이다.
프로젝트 안에서의 역할(1차/2차/3차)에 따라 구조가 달라질 수 있다.

---

# 2. 반드시 만들어야 하는 화면 범위

이번 설계에서는 아래 화면들만 만든다.

## A. 로그인 후 첫 화면

* 협력사가 로그인하면 협력사 포털 메인 레이아웃으로 진입한다
* 이 메인 레이아웃에는 **전역 네비게이션1**이 보인다
* 로그인 직후 기본 활성 화면은 **네비게이션1의 프로젝트 탭 화면**이어야 한다
* 즉, 로그인 직후 첫 화면은 곧 **프로젝트 선택 화면**이다
* 아직 특정 프로젝트에 들어가기 전 상태이므로, 이 시점에는 **네비게이션2가 없다**

## B. 프로젝트 선택 화면

* 네비게이션1의 **프로젝트 탭**에서 보이는 화면이다
* 협력사가 참여 중인 프로젝트 목록을 보여준다
* 각 프로젝트는 카드 또는 리스트 형태로 보여준다
* 사용자는 여기서 어떤 프로젝트로 들어갈지 선택한다
* 프로젝트마다 해당 협력사가 **1차 / 2차 / 3차 중 어떤 역할인지 명확히 보이게** 하라

## C. 프로젝트 진입 후 화면

* 사용자가 프로젝트 선택 화면에서 특정 프로젝트를 선택하면
* 그때부터 프로젝트 컨텍스트 안으로 진입한다
* 이때 전역 네비게이션1은 유지되고,
* 별도로 **프로젝트 전용 네비게이션2**가 추가로 나타난다

이번 단계에서는 아래 3가지 경우를 각각 분리해서 구조를 제안하라.

* 프로젝트에 **1차 협력사**로 들어간 경우
* 프로젝트에 **2차 협력사**로 들어간 경우
* 프로젝트에 **3차 협력사**로 들어간 경우

중요:
이번 단계에서는 1차/2차/3차 각각의 경우에 대해
**네비게이션2의 탭 구조와 프로젝트 내부 화면 골격까지만 설계**하라.
각 탭 안의 상세 콘텐츠는 만들지 마라.

# 3. 네비게이션1 설계 원칙

네비게이션1은 프로젝트와 무관한 **전역 메뉴**여야 한다.
로그인 후 항상 동일하게 보이며, 프로젝트 진입 전/후 모두 유지된다.

반드시 포함할 메뉴:

* **프로젝트**
* **회사 프로필**
* **설정**

선택적으로 포함 가능:

* 조직/사용자 관리
* 알림

단, 이번 설계에서는 최소한 아래 3개는 반드시 보여라.

* 프로젝트
* 회사 프로필
* 설정

## 네비게이션1의 핵심 역할

* 협력사가 참여 중인 프로젝트를 고르는 진입점 제공
* 회사 차원의 전역 정보 관리
* 프로젝트 내부 업무 영역과 전역 영역을 구분하는 기준축

---

# 4. 프로젝트 선택 화면 설계 원칙

프로젝트 선택 화면은 단순한 목록이 아니라
**“내가 어떤 역할로 어떤 프로젝트에 들어갈지 고르는 런처 화면”**이어야 한다.

각 프로젝트 카드/행에는 아래 정보가 보여야 한다.

* 프로젝트명
* 상위차사 또는 고객사명
* 납품 항목 또는 제품명
* 계약번호
* 현재 이 프로젝트에서의 역할: **1차 / 2차 / 3차**
* 진행 상태
* 마감일
* 최근 업데이트 일시

프로젝트 역할이 한눈에 구분되도록
**1차 / 2차 / 3차 역할 배지**를 명확하게 표시하라.

예:

* 1차 배지
* 2차 배지
* 3차 배지

프로젝트를 누르면
그 프로젝트의 역할에 맞는 **네비게이션2가 포함된 내부 프로젝트 화면**으로 진입해야 한다.

---

# 5. 프로젝트별 권한/운영 전제

이번 설계에서 다음 정책을 반드시 반영하라.

## 1차 협력사

* 자기 하위 공급망 전체를 관리하는 주체
* 하위차사 구조를 볼 수 있다
* 하위차사 데이터를 볼 수 있다
* **하위차사 데이터 직접 수정 가능**
* 최종 취합 후 원청에 제출하는 책임이 있다

## 1차가 아닌 협력사(2차, 3차)

* 자기 하위 공급망 구조를 볼 수 있다
* 자기 하위차사 데이터를 볼 수 있다
* **하위차사 데이터 직접 수정 불가**
* 대신 **수정 요청 / 보완 요청**만 가능하다
* 자기 구간 결과를 상위차사에 제출한다

즉,
2차와 3차도 하위 구조를 볼 수는 있지만
1차와 달리 **운영 관리자 권한은 없다**는 점을 반영하라.

---

# 6. 네비게이션2 설계 원칙

이번 단계에서는
1차 / 2차 / 3차 각각에 대해 **네비게이션2 탭 이름과 구조만 설계**하라.

## 중요 설계 원칙

* 1차의 네비게이션2는 **관리형 구조**
* 2차와 3차의 네비게이션2는 **조회/입력 중심 구조**
* 다만 2차와 3차도 하위 공급망 구조를 볼 수는 있어야 한다
* 차이는 **직접 수정 권한 유무**에서 발생한다
* 이번 단계에서는 상세 기능을 깊게 만들지 말고, **탭 이름과 구조만 정의**하라

---

# 7. 1차인 경우의 네비게이션2

프로젝트를 1차 역할로 선택했을 때 보이는 네비게이션2를 설계하라.

1차는 하위 공급망 전체를 관리하고 최종 취합/제출하는 입장이므로,
프로젝트 내부 네비게이션은 **관리자형 워크스페이스**처럼 보여야 한다.

권장 탭 구조:

* **대시보드**
* **공급망 관리**
* **데이터 관리**
* **데이터 조회**
* **PCF 산정/전송**
* **이력**

이번 단계에서는 위 탭들을 중심으로
좌측 사이드바 또는 상단 탭 구조 등
일관된 프로젝트 내부 네비게이션 구조를 제시하라.

---

# 8. 2차인 경우의 네비게이션2

프로젝트를 2차 역할로 선택했을 때 보이는 네비게이션2를 설계하라.

2차는 자사 데이터 입력과 상위 제출이 핵심이며,
자기 하위 공급망은 볼 수 있지만 직접 수정하지는 못한다.

따라서 2차 프로젝트 내부 구조는
**조회/입력/요청 중심 워크스페이스**로 설계하라.

권장 탭 구조:

* **대시보드**
* **공급망 조회**
* **데이터 관리**
* **데이터 조회**
* **PCF 산정/전송**
* **이력**

중요:

* 공급망 **관리**가 아니라 공급망 **조회**
* 하위 데이터 **직접 수정**이 아니라 **수정요청/보완요청** 중심이라는 전제를 반영하라
* 이번 단계에서는 탭 구조만 만들고 상세 페이지 내용은 만들지 마라

---

# 9. 3차인 경우의 네비게이션2

프로젝트를 3차 역할로 선택했을 때 보이는 네비게이션2를 설계하라.

3차도 2차와 마찬가지로
기본적으로는 자사 입력과 상위 제출 중심이다.
단, 프로젝트 구조상 하위가 존재할 수 있다면 공급망 구조를 조회할 수 있어야 한다.

권장 탭 구조:

* **대시보드**
* **공급망 조회**
* **데이터 관리**
* **데이터 조회**
* **PCF 산정/전송**
* **이력**

즉, 3차도 2차와 유사한 프로젝트 네비게이션2를 갖되,
프로젝트 헤더나 역할 배지를 통해
현재 사용자가 **3차로 참여 중임을 명확히 보여라.**

# 10. 프로젝트 진입 후 공통 헤더 원칙

프로젝트에 들어간 뒤 상단 헤더 또는 breadcrumb 영역에는
아래 정보가 공통적으로 보여야 한다.

* 현재 프로젝트명
* 상위차사 또는 제출 대상
* 현재 내 역할: 1차 / 2차 / 3차
* 계약번호 또는 납품 단위 식별값

중요:
탭 내부 콘텐츠를 설계하지 않더라도
현재 사용자가 **어떤 프로젝트에 어떤 차수로 들어와 있는지**는
항상 명확하게 보여야 한다.

# 11. 이번 단계에서 절대 하지 말 것

아래는 이번 작업 범위에서 제외한다.

* 각 탭 내부의 상세 폼 설계
* 테이블 컬럼 상세 정의
* 구체적인 입력 필드 설계
* 계산 로직 상세 UI
* 데이터 카드/차트 상세내용
* 팝업/모달 상세 설계

즉, 지금은 **IA / 네비게이션 / 화면 골격 설계만** 하라.

# 12. 최종 산출물 요구사항

다음 5가지를 한 세트로 제안하라.

## 1) 로그인 후 보이는 협력사 포털 메인 레이아웃

* 전역 네비게이션1 포함
* 기본 활성 화면은 프로젝트 탭

## 2) 네비게이션1의 프로젝트 탭 화면

* 프로젝트 선택 화면
* 1차 / 2차 / 3차 프로젝트가 한눈에 구분되는 구조

## 3) 1차 프로젝트 진입 시 네비게이션2 구조

## 4) 2차 프로젝트 진입 시 네비게이션2 구조

## 5) 3차 프로젝트 진입 시 네비게이션2 구조

중요:

* 1차 / 2차 / 3차의 **구조 차이**가 분명히 보이게 하라
* 그러나 전체 디자인 시스템은 하나의 협력사 포털처럼 일관되게 유지하라
* 세부 콘텐츠보다 **정보구조와 화면 전환 흐름**을 우선하라

* `#F6F8FB` — 섹션 구분용 배경
* `#0F172A` — 텍스트 강조(짙은 네이비)
* `#8C8C8C` — 설명문, 서브 텍스트용

### Gradient Example

* `linear-gradient(90deg, #5B3BFA 0%, #00B4FF 100%)`

## 13-2. Typography

### 기본 폰트

* 한글: `Pretendard` 또는 `SUIT`
* 영문: `Inter` 또는 `Poppins`

### 폰트 두께

* 제목: `700 (Bold)`
* 부제/버튼: `600 (Semi-Bold)`
* 본문: `400 (Regular)`
* 설명문: `300 (Light)`

### 폰트 크기 체계

* 메인 헤드라인: `42~48px`
* 섹션 타이틀: `28~32px`
* 서브 타이틀: `20~22px`
* 본문: `16~18px`
* 버튼 텍스트: `16px`
* 캡션 / 주석: `14px`

이번 작업은 상세 UI가 아니라 구조 설계 단계이므로,
위 타이포그래피 체계를 **화면 위계감과 제목/탭/라벨 스타일 기준**으로만 반영하라.

## 13-3. 컴포넌트 스타일 가이드

### 버튼

* 기본 버튼: 파란-보라 그라디언트 배경 + 흰색 텍스트
* Gradient: `linear-gradient(90deg, #5B3BFA, #00B4FF)`
* Hover 시 밝기 10% 증가
* Radius: `12~16px`
* Shadow: `0px 4px 12px rgba(91, 59, 250, 0.2)`

### 카드

* 배경: `#FFFFFF`
* Border-radius: `20px`
* 그림자: `0px 4px 16px rgba(0,0,0,0.05)`
* 내부 패딩: `24~32px`

### 네비게이션 바

* 상단 고정 또는 좌측 고정 구조 모두 가능
* 배경은 반투명 흰색 `rgba(255,255,255,0.8)` 또는 매우 밝은 중성 톤 사용
* active 메뉴 하이라이트 색상: `#5B3BFA`

이번 단계에서는 상세 컴포넌트를 설계하기보다,
**네비게이션과 화면 골격이 이 스타일 가이드를 따르도록** 하라.
