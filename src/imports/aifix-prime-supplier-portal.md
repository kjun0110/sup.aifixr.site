너는 제조업 공급망 ESG/PCF 관리 SaaS AIFIX의 시니어 프로덕트 디자이너다.
현재 화면은 **협력사 포털의 1차 협력사(Prime Supplier)**가 프로젝트에 진입했을 때 보이는 네비게이션2 기반 UI이며, 1차는 2차~N차 하위차사를 모두 조회/관리하고, 필요 시 하위 데이터 직접 수정 및 최종 취합 후 원청 제출까지 수행한다.

아래 요구사항에 맞게 기존 화면을 “구조 유지 + 문구/필드/상태/정보구조 개선” 중심으로 수정하라.
(세부 입력 필드 설계/새 테이블 컬럼 과다 추가 금지. 핵심 요소만 정리.)

0) 공통 수정 (전체 탭 공통 규칙)

프로젝트 헤더(상단 고정 영역)에 다음을 항상 노출:

프로젝트명 / 내 차수(1차) / 제출 대상(원청) / 계약번호 / 마감일 / 현재 상태

상태 체계를 전 탭에서 동일 용어로 통일:

미제출 / 제출완료 / 보완요청 / 검증실패
(기존 “완료/대기/진행” 등 혼재된 표현이 있으면 위 4개 기준으로 매핑)

1) 대시보드 탭 수정
1-1. KPI 카드 명칭/의미 정합성 수정

“전체 데이터 제출률” → **“하위 협력사 제출률”**로 변경
(1차 관점에서 전체=내 아래 공급망이므로 명확히)

“대기 중 작업” → “조치 필요” 또는 **“확인 필요 작업”**으로 변경
(실제 의미가 보완/확인 요청이므로 ‘대기’ 표현 제거)

“완료된 작업” → “완료된 제출” 또는 **“완료된 데이터 항목”**으로 변경
(PCF는 “업무 종료”가 아니라 “제출/데이터 단위 완료”가 더 정확)

1-2. 대시보드 내 리스트/바로가기 정리

“미제출/지연 업체” 리스트는 유지하되, row 액션을 다음처럼 표준화:

보완요청 (메시지/기한 포함)

수정하기(직접 수정) (1차 권한)

상세 보기 (조회 전용)

상단 경고 배너는 유지하되, CTA는 “미제출 업체 보기”처럼 리스트로 점프하도록 연결

2) 공급망 관리 탭 수정
2-1. 테이블 컬럼에서 ‘상위업체’ 제거(또는 축소)

1차 화면에서 “상위업체”는 대부분 ‘우리회사(1차)’로 고정되어 의미가 약함.
→ 협력사 리스트 테이블에서 상위업체 컬럼을 제거하거나, 꼭 필요하면 “상위업체”를 기본 숨김(세부 패널에서만 표기)으로 바꿔라.

추천 테이블 컬럼(간결 유지):

업체명 / 차수 / 상태 / 최근 제출일 / 마감 / 액션

2-2. 트리 + 리스트 + 상세패널의 역할 분리 강화

좌측: 공급망 트리(구조 탐색 전용)

중앙: 협력사 리스트(업체 목록/상태/마감 중심)

우측: 상세 패널(선택 업체의 정보/요청이력/액션)

“협력사를 선택하면 상세 정보가 표시됩니다” 빈상태 문구 유지

2-3. 1차 전용 액션 명확화(직접 수정/요청의 구분)

상세 패널 또는 리스트 액션에서 버튼 그룹을 다음처럼 구분 노출:

직접 수정(1차 전용): 하위 제출 데이터를 즉시 편집

보완요청: 하위차사에게 요청(메시지/마감 포함)

재요청: 미응답/지연 업체에 리마인드
(“수정요청”이라는 표현은 1차에서는 ‘직접 수정’과 혼동되므로, 1차 화면에서는 보완요청/재요청 중심으로 용어를 정리)

3) 데이터 관리 탭 수정
3-1. 버튼 순서/문구 수정

우상단 액션 버튼 순서를 아래로 재배치:

임시 저장 → 검증 → 저장
(작성→검증→저장 흐름과 일치)

3-2. “상세 편집” 버튼 문구 개선

각 카드의 “상세 편집” → “입력/수정” 또는 **“데이터 입력”**으로 변경
(사용자가 해야 하는 행동을 더 직접적으로 표현)

4) 데이터 조회 탭 수정
4-1. ‘내부 기준 v2.3’ 용어 개선

필터에서 “내부 기준 v2.3” → “데이터 버전(v2.3)” 또는 **“입력 버전(v2.3)”**로 변경
(사용자가 의미를 바로 이해하게)

4-2. 상단 세부탭 유지(구조는 이미 정답)

“입력 데이터 조회” / “PCF 결과 조회” 2개 탭 분리는 유지

조회 화면에는 편집/저장 액션을 절대 노출하지 말 것

5) PCF 산정/전송 탭 수정
5-1. Functional Unit(단위 기준) 표시 추가 (필수)

“총 PCF 1102 kgCO2e” 표기 옆/아래에 **기준 단위(Functional Unit)**를 명확히 추가:

예: 1102 kgCO2e / 제품(또는 1kg, 1batch 등)
(PCF 결과는 단위 기준이 없으면 개념적으로 불완전)

5-2. 결과 버전/스냅샷 연결성 강조

“계산 실행/재계산”으로 생성된 결과는 버전으로 저장되고

“데이터 조회 > PCF 결과 조회”에서 재열람 가능하다는 안내를 유지/강화

6) 이력 탭 수정
6-1. ‘영향 범위’ 필드(또는 라벨) 추가 (권장)

이력 리스트 row에 “영향 범위”를 추가하거나, 상세 패널에 반드시 표시:

예: 자사 데이터 / 2차 협력사 / 전체 계산
(PCF는 변경이 어디에 영향을 줬는지 추적이 중요)
