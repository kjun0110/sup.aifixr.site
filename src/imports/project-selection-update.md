너는 AIFIX 협력사 포털의 시니어 프로덕트 디자이너다.

현재 ‘프로젝트 선택’ 화면에서 1차 프로젝트 카드에
‘상위차사: 원청사 A’, ‘상위차사: 원청사 D’처럼 서로 다른 원청사가 표시되고 있다.

하지만 우리 서비스의 전제는 **‘협력사 포털 = 단일 원청 전용 포털’**이므로
프로젝트 카드에서 서로 다른 원청사가 표시되는 것은 논리적으로 불가능하다.

따라서 다음 원칙에 따라 **프로젝트 선택 화면을 수정하라.**

---

# 1. 원청사 단일 고정 원칙

이 협력사 포털은 **단일 원청 전용 포털**이므로
포털 전체에서 원청사는 **항상 동일한 하나의 회사**여야 한다.

따라서 다음 규칙을 적용한다.

* 프로젝트 카드에서 서로 다른 원청사가 등장하면 안 된다.
* 1차 프로젝트의 상위차사는 항상 동일한 **단일 원청사 이름**으로 표시된다.
* 예시

  * 상위차사: 원청사 A
  * 상위차사: 원청사 A
  * 상위차사: 원청사 A

즉, 1차 프로젝트 카드에서 원청사가 프로젝트마다 달라지지 않도록 수정한다.

---

# 2. 프로젝트 카드 정보 라벨 통일

프로젝트 카드의 구조를 단순화하기 위해
**모든 차수에서 동일한 라벨을 사용한다.**

기존 라벨:
상위차사

이 라벨을 **모든 프로젝트 카드에서 동일하게 유지한다.**

차수별 표시 규칙은 다음과 같다.

### 1차 프로젝트

상위차사 = 원청사

예시
상위차사: 원청사 A

### 2차 프로젝트

상위차사 = 1차 협력사

예시
상위차사: 1차 협력사 B

### 3차 프로젝트

상위차사 = 2차 협력사

예시
상위차사: 2차 협력사 C

즉, 프로젝트 카드의 정보 구조는 다음처럼 **완전히 동일한 형태**를 유지한다.

상위차사
납품 항목
계약번호
마감일
최근 업데이트

라벨을 차수별로 다르게 만들지 말고 **모든 카드에서 동일한 구조를 유지하라.**

---

# 3. 검색창 placeholder 수정

현재 검색 placeholder에 다음 표현이 포함되어 있다.

"프로젝트명, 계약번호, 고객사명으로 검색"

하지만 이 포털은 단일 원청 포털이므로
**고객사명 검색은 필요하지 않다.**

따라서 placeholder를 아래처럼 수정한다.

"프로젝트명, 계약번호, 납품 항목으로 검색"

---

# 4. 기존 UI 레이아웃 유지

다음 요소들은 변경하지 않는다.

* 카드 레이아웃
* 카드 디자인 스타일
* 카드 배치 구조
* 색상 및 디자인 시스템
* 프로젝트 카드 UI 컴포넌트

수정 대상은 다음 **텍스트 구조와 데이터 규칙**만이다.

* 원청사 단일 고정
* 상위차사 표기 통일
* 검색 placeholder 수정

---

# 5. 작업 범위 제한

이번 수정 작업에서는 다음 작업을 하지 않는다.

* 새로운 UI 컴포넌트 추가
* 카드 레이아웃 재설계
* 프로젝트 상세 화면 변경
* 네비게이션 구조 변경
* 데이터 구조 확장

오직 **프로젝트 선택 화면의 카드 정보 구조와 텍스트 규칙만 수정한다.**
