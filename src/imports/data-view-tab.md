너는 제조업 공급망 ESG/PCF 관리 SaaS **AIFIX 협력사 포털**의 시니어 프로덕트 디자이너다.

현재 설계 대상은:
협력사 포털 → 프로젝트 선택 → **비1차(2차/3차) 역할 프로젝트 진입 후 화면(네비게이션2)** 중
**[데이터 조회] 탭**이다.

이번 작업은 **비1차 전용 화면**이며 1차 화면은 설계하지 않는다.

========================
[고정 전제]
========================
- 협력사 포털은 단일 원청 전용 포털이다(원청이 여러 개로 보이면 안 됨).
- 협력사는 구매/PCF 직무를 분리하지 않고 **통합 템플릿 1개**로 입력한다.
- 비1차 권한:
  - 하위차사 구조/데이터 조회 가능
  - 하위 데이터 **직접 수정 불가**
  - 하위 데이터에 대해 **수정요청/보완요청만 가능**
  - 자기 구간 결과를 **상위차사**에 제출
- 핵심 업무 흐름:
  1) 데이터 관리 = 입력/수정(계산 없음)
  2) PCF 산정·전송 = 계산/검토/전송
  3) 데이터 조회 = 입력값/결과값 확인 (**반드시 분리해서 보여주기**)

========================
[공통 레이아웃]
========================
- 상단 고정 “프로젝트 컨텍스트 헤더”(모든 탭 공통)
  - 프로젝트명 / 내 차수(2차 또는 3차) / 제출 대상(상위차사명) / 계약번호 / 마감일 / 상태
- 그 아래 네비게이션2 탭바(전체 탭은 표시만, 상세 설계는 하지 않음):
  - 대시보드 / 공급망 조회 / 데이터 관리 / **데이터 조회(현재)** / PCF 산정·전송 / 이력

========================
[데이터 조회 탭 설계 목표]
========================
비1차 사용자가 아래를 **“조회 전용”**으로 확인하게 한다.
- 내가 입력한 값(원천 데이터)
- 시스템이 계산한 값(PCF 결과)
- 하위차사에서 올라온 값(참조)
- 상위 제출에 쓰인 스냅샷/버전

중요:
- 데이터 조회 탭에서는 **편집/저장/직접 수정** 액션을 절대 노출하지 말 것
- 대신 “수정요청/보완요청”은 **조회 맥락에서만** 보조 액션으로 허용(하위 데이터에 한함)

========================
[핵심 구조: 상단 세부 탭 2개로 반드시 분리]
========================
데이터 조회 탭 상단에 세부 탭 2개를 만든다.

1) **입력 데이터 조회**
2) **PCF 결과 조회**

이 분리는 필수이며, 한 화면에서 섞어 보여주지 않는다.

------------------------------------------------
1) 입력 데이터 조회 (원천 데이터 전용)
------------------------------------------------
목적:
- 자사 입력값과 하위차사 제출값(참조)을 “원천 데이터”로 확인

레이아웃(권장: 상단 필터 바 + 본문 테이블/리스트 + 우측 상세 패널)

A. 상단 필터 바
- 데이터 범위: 자사 / 하위차사(참조) / 전체
- 업체 선택(하위 포함)
- 차수(2차/3차 및 하위 단계)
- 제출 상태(미제출/제출/검증실패 등)
- 기간(업데이트 기준)
- 버전(원천 데이터 버전 또는 최신/전송본 기준)
- 검색: 항목명/업체명/계약번호/납품항목

B. 본문 리스트(테이블)
- 기본 컬럼 예:
  - 항목명(또는 섹션)
  - 값 요약
  - 데이터 출처(자사 입력 / 하위 제출)
  - 상태(저장됨/미완성/검증실패 등)
  - 마지막 업데이트
- 행 클릭 시 우측 상세 패널 열림

C. 우측 상세 패널(조회 전용)
- 선택 항목의 상세 정보:
  - 값(원문)
  - 출처(자사/하위 업체명)
  - 적용된 단위/메타
  - 관련 버전/시간
- 허용 액션(비1차 규칙 반영):
  - 자사 데이터인 경우: “데이터 관리로 이동(편집)” 링크만 제공(직접 편집은 이 탭에서 금지)
  - 하위 데이터인 경우: “수정요청” / “보완요청” 버튼 제공(직접 수정 금지)
    → 클릭 시 요청 모달(메시지 입력) 표시

※ 금지:
- 이 화면에 PCF 계산 결과(총 PCF 등)를 보여주지 말 것
- 저장/수정 버튼 금지(데이터 관리 탭에서만)

------------------------------------------------
2) PCF 결과 조회 (계산 결과 전용)
------------------------------------------------
목적:
- PCF 산정·전송 탭에서 생성된 “계산 결과”를 버전 기반으로 조회/비교

레이아웃(권장: 상단 버전 선택 + 결과 요약 카드 + 상세 리스트 + 비교 패널)

A. 상단 “결과 버전 선택 바”
- 버전 드롭다운(예: 계산 버전 v1, v2 / 전송본 스냅샷)
- 기준 선택: 최신 계산 / 전송본 / 특정 날짜
- 비교 모드 토글: 버전 A vs 버전 B

B. 결과 요약 카드(큰 숫자 중심)
- “내 구간 PCF 결과” 요약(총합)
- 하위 반영 현황(반영 완료/부분/미반영, %)
- 경고 수(누락/미제출/이상치 등)
- 마지막 계산 시각 / 계산 상태

C. 결과 상세 영역(테이블 또는 카드 리스트)
- 항목별 합산 결과(큰 구조만)
- 하위차사 반영 결과(업체별 요약)
- 경고/주의 항목 리스트(조치가 필요한 항목)

D. 버전 비교 패널(비교 모드 시)
- 버전 A vs B 변화 요약 카드:
  - 총 PCF 변화(+/-)
  - 주요 기여 항목 변화 Top N
  - 경고 변화(증가/감소)
- 상세 수치 테이블을 길게 만들지 말고 “변화 요약” 중심

E. 허용 액션(조회/추적 중심)
- “PCF 산정·전송 탭으로 이동” (재계산/전송은 그 탭에서)
- “이력 보기” (해당 버전의 계산/전송 이벤트로 점프)

※ 금지:
- PCF 결과를 이 화면에서 수정/재계산 실행하는 버튼 금지
- 하위 데이터 직접 수정 금지

========================
[모달/요청 UX(비1차 공통)]
========================
입력 데이터 조회에서 하위 데이터를 선택했을 때만 허용:
- 수정요청 모달
- 보완요청 모달
모달 구성(구조만):
- 요청 유형
- 요청 메시지
- 마감 기한(선택)
- 전송 버튼

========================
[산출물]
========================
Figma에서 아래 1개 탭(데이터 조회) 안에:
- 상단 세부 탭 2개(입력 데이터 조회 / PCF 결과 조회)
- 각 탭의 레이아웃 골격(필터 바, 리스트/테이블, 상세 패널, 버전 선택/비교 구조)
- 비1차 권한(직접 수정 불가)과 흐름(입력→산정→조회) 규칙이 명확히 드러나게 설계

세부 입력 필드/테이블 컬럼 디테일 정의는 하지 말고,
**구성요소 배치와 화면 전환/액션 가능 범위 중심**으로 설계하라.
