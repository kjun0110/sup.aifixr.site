너는 제조업 공급망 ESG/PCF 관리 SaaS **AIFIX 협력사 포털**의 시니어 프로덕트 디자이너다.

이번 작업 범위는 “협력사 포털 > 프로젝트 선택 > 1차 역할 프로젝트 진입 후” 화면 중
네비게이션2 탭 **[대시보드]**와 **[공급망 관리]** 2개 탭만 구체적으로 설계하는 것이다.
(비1차 화면은 만들지 말 것)

[고정 전제]
- 포털은 단일 원청 전용이다.
- 현재 프로젝트에서 사용자는 “1차 협력사” 역할이다.
- 1차 권한: 하위차사 구조/데이터 조회 가능 + 하위 데이터 직접 수정 가능 + 최종 취합 후 원청 제출
- 협력사는 직무(구매/PCF)를 분리하지 않고 통합 템플릿에 입력한다.
- 핵심 흐름: 데이터 관리(입력) → PCF 산정/전송(계산/전송) → 데이터 조회(확인)
  단, 이번 작업에서는 “대시보드/공급망 관리”만 설계한다.

[공통 레이아웃 규칙]
- 상단에는 프로젝트 컨텍스트 헤더를 고정으로 넣어라(모든 탭 공통):
  프로젝트명 / 내 차수(1차) / 제출 대상(원청사명) / 계약번호 / 마감일 / 상태
- 네비게이션2 탭은 상단 탭바 형태로 유지(대시보드, 공급망 관리, 데이터 관리, 데이터 조회, PCF 산정/전송, 이력)
- 디자인 톤: B2B SaaS, 신뢰감/정돈된 레이아웃, 카드 기반, 여백 충분히

=====================================================
A) [대시보드] 탭 (1차 운영형 대시보드) 구체 설계
=====================================================
목표: 1차 협력사가 “전체 하위 공급망 진행 상황을 운영/통제”하고, 바로 행동(요청/수정/이동)할 수 있게 한다.
대시보드는 ‘프로젝트 운영센터’ 느낌이어야 한다.

1) 상단 KPI 카드 4개(가로 4열)
- 전체 데이터 제출률(%)
- 대기 중 작업(건)
- 완료된 작업(건)
- 보완 요청(건)
각 카드에는 “상태 아이콘/미니 트렌드/설명 한 줄”을 넣어 가독성 확보

2) 중단 좌측: “미제출/지연 업체” 액션 리스트(테이블 or 리스트 카드)
- 제목: 미제출/지연 업체
- 행 정보 최소: 업체명, 차수(2차/3차), 현재 상태(미제출/보완필요/지연), 최근 제출일, 마감까지 D-day
- 행 우측 CTA 버튼 2개를 반드시 제공:
  (1) 보완요청/재요청 버튼
  (2) 데이터 보기/수정으로 이동 버튼
- 리스트 상단 필터: 차수 필터(2차/3차/전체), 상태 필터(미제출/보완필요/지연)

3) 중단 우측: “공급망 단계별 진행 현황” 요약(스텝/막대/도넛 중 택1)
- 단계(2차/3차) 또는 상태(미제출/제출완료/보완요청/검증실패)로 분포를 보여준다
- 각 구간을 클릭하면 ‘공급망 관리’ 탭으로 이동하거나 리스트가 필터링되는 상호작용을 암시(주석으로 표기)

4) 하단: “빠른 작업(Quick Actions)” 패널(큰 버튼 3개)
- 공급망 관리로 이동
- 데이터 관리로 이동
- PCF 산정/전송으로 이동
각 버튼에는 한 줄 설명을 넣어 ‘왜 눌러야 하는지’가 보이게 한다.

5) 경고/알림 영역(상단 또는 하단 배너)
- 마감 임박, 미제출 업체 존재, 누락/검증 실패 등 ‘제출 리스크’ 신호를 배너로 표시
- 배너 CTA: “미제출 업체 보기”, “검증 실패 항목 확인” 등

[대시보드 금지사항]
- 대시보드에서 PCF 계산 결과(총 PCF 숫자)를 메인 콘텐츠로 노출하지 말 것
- 입력 폼을 대시보드에 넣지 말 것(대시보드는 운영/이동/액션 중심)

=====================================================
B) [공급망 관리] 탭 (1차 전용: 트리 + 운영/수정) 구체 설계
=====================================================
목표: 1차가 자기 하위 공급망 “전체 트리 구조”를 보고,
하위 업체의 상태를 관리하며, 필요 시 하위 데이터까지 “직접 수정”할 수 있게 한다.

1) 화면을 3영역 레이아웃으로 설계(좌/중/우)
- 좌측(고정 사이드): 공급망 트리(Company Tree)
- 중앙: 선택 노드 기준 리스트/테이블(업체 목록)
- 우측(슬라이드 패널): 업체 상세/액션 패널

2) 좌측 트리 영역(공급망 트리)
- 루트: 현재 사용자(1차) 고정 표시
- 하위: 2차 → 3차 계층을 들여쓰기/연결선으로 명확히 표시
- 각 노드에 상태 배지 표시(미제출/제출완료/보완요청/검증실패 등)
- 트리 상단 컨트롤:
  - 검색(업체명)
  - 펼치기/접기
  - 상태 필터(미제출/보완필요 등)

3) 중앙 리스트/테이블 영역(업체 운영 리스트)
- 테이블 컬럼 최소:
  업체명 / 차수 / 상위업체 / 상태 / 최근 제출일 / 마감 D-day / 보완요청 여부
- 행 클릭 시 우측 상세 패널이 열리며 해당 업체 컨텍스트를 보여준다
- 상단 탭(선택): “전체” / “미제출” / “보완필요” / “검증실패” 퀵필터 제공
- 리스트 상단에는 ‘선택된 트리 노드(필터 컨텍스트)’가 보이도록 breadcrumb/칩으로 표시

4) 우측 상세 패널(운영 + 수정 액션의 핵심)
선택한 업체(2차/3차)의 상세 패널에는 다음 섹션을 포함:

(1) 업체 요약
- 업체명, 차수, 상위업체, 납품항목(있다면), 담당자, 연락수단(선택)
- 현재 상태 + 마지막 제출일 + 마감 D-day

(2) 제출/요청 이력 미니 타임라인
- 최근 5건: 제출, 보완요청, 재요청, 수정, 검증실패 등 이벤트
- “전체 이력 보기” 링크(이력 탭으로 이동)

(3) 주요 액션 버튼(1차 전용 권한 반영)
- 데이터 보기(조회 탭 또는 데이터 조회 상세로 이동)
- 데이터 직접 수정(데이터 관리 > 하위차사 데이터 편집으로 이동)
- 보완요청 보내기(모달/패널로 메시지/사유 입력 UI 골격)
- 재요청 보내기(마감/기한 설정 UI 골격)
※ 비1차처럼 “수정요청만”이 아니라, 반드시 “직접 수정” 버튼이 존재해야 한다.

(4) 상태 변경/관리(선택)
- 상태를 강제로 바꾸는 기능은 최소화하되,
  “보완요청 발송됨/응답대기” 같은 운영 상태는 표시(칩 형태)

5) 공급망 관리 탭의 핵심 UX 원칙(주석으로 명시)
- 1차는 하위 데이터에 대해 “직접 수정 가능”
- 따라서 공급망 관리에서 ‘운영 → 즉시 조치’가 가능해야 함
- 트리(구조 파악) + 리스트(대상 선별) + 패널(조치 실행) 3단 구조로 빠르게 처리

[공급망 관리 금지사항]
- 하위 데이터 편집 화면(실제 입력 폼)은 여기에서 상세 구현하지 말고
  “데이터 직접 수정”은 데이터 관리 탭으로 이동하는 흐름만 제공(버튼/전환 관계만)
- PCF 계산 버튼을 공급망 관리에 넣지 말 것(계산은 PCF 산정/전송 탭)

=====================================================
산출물
=====================================================
- 1차 프로젝트 진입 후 화면에서:
  1) 대시보드 탭 1개 화면(구성요소 배치 + CTA/필터 포함)
  2) 공급망 관리 탭 1개 화면(트리/리스트/상세패널 + 1차 전용 액션 포함)
- 두 화면 모두 상단 프로젝트 컨텍스트 헤더와 네비게이션2 탭바를 포함
- 세부 입력 필드/테이블 컬럼 상세 정의는 최소 수준으로만(구조 중심)
