너는 제조업 공급망 ESG/PCF 관리 SaaS **AIFIX 협력사 포털**의 시니어 프로덕트 디자이너다.

현재 설계 대상은

협력사 포털 → 프로젝트 선택 → **비1차(2차/3차) 역할 프로젝트 진입 후 화면(네비게이션2)** 중  
**데이터 관리 탭**이다.

이 화면은 **비1차 협력사용 화면이며 1차 협력사 화면은 설계하지 않는다.**

--------------------------------
[시스템 전제]
--------------------------------

협력사 포털은 다음 구조를 가진다.

- 단일 원청 전용 포털
- 협력사는 구매/PCF 직무를 분리하지 않고 **통합 템플릿 하나에 데이터 입력**

현재 사용자는 **비1차 협력사 (2차 또는 3차)** 이다.

비1차 협력사 권한

- 하위차사 구조 조회 가능
- 하위차사 데이터 조회 가능
- 하위 데이터 **직접 수정 불가**
- 하위 업체에 **수정요청 / 보완요청만 가능**
- 자기 구간 PCF 결과를 **상위차사에게 제출**

--------------------------------
[업무 흐름 규칙]
--------------------------------

협력사 시스템의 업무 흐름

1️⃣ 데이터 관리  
→ 데이터 입력 / 수정 (계산 없음)

2️⃣ PCF 산정·전송  
→ PCF 계산 및 상위차사 제출

3️⃣ 데이터 조회  
→ 입력 데이터 및 계산 결과 확인

따라서 **데이터 관리 탭은 입력/수정 전용 화면이며 계산 결과는 노출하지 않는다.**

--------------------------------
[공통 화면 구조]
--------------------------------

화면 상단에는 **프로젝트 컨텍스트 헤더**가 고정 표시된다.

헤더 정보

- 프로젝트명
- 내 차수 (2차 또는 3차)
- 제출 대상 (상위차사명)
- 계약번호
- 마감일
- 상태

그 아래에 **네비게이션2 탭**이 표시된다.

네비게이션2 탭

- 대시보드
- 공급망 조회
- 데이터 관리 (현재 화면)
- 데이터 조회
- PCF 산정·전송
- 이력

--------------------------------
[데이터 관리 탭 목적]
--------------------------------

데이터 관리 탭은 **데이터 입력과 수정이 이루어지는 유일한 화면**이다.

비1차 협력사에게 필요한 기능

- 자사 데이터 입력 및 수정
- 하위차사 제출 데이터 확인
- 하위 업체에 수정요청 / 보완요청

주의

비1차 협력사는 **하위 데이터를 직접 수정할 수 없다.**

--------------------------------
[화면 레이아웃]
--------------------------------

데이터 관리 탭 내부에는 **상단 세부 탭 2개**를 만든다.

세부 탭

- 자사 데이터
- 하위차사 참조

--------------------------------
1️⃣ 자사 데이터 탭
--------------------------------

목적

현재 협력사가 **직접 입력하는 데이터 관리 화면**

레이아웃

상단

상태 표시 카드

카드 내용

- 입력 진행률
- 검증 상태
- 마지막 저장 시간

그 아래

**통합 템플릿 입력 영역**

구조

- 여러 섹션 카드 형태
- 각 섹션에는 입력 폼 포함

예시 섹션

- 기업/조직 정보
- 납품 제품 정보
- 활동 데이터
- 에너지 사용 데이터
- 기타 환경 데이터

각 섹션에는 다음 버튼 제공

- 저장
- 임시 저장
- 검증 실행

--------------------------------
입력 상태 표시
--------------------------------

각 섹션에는 상태 표시 배지 제공

예

- 저장됨
- 미완성
- 검증 실패
- 수정 필요

사용자가 입력 상태를 한눈에 확인할 수 있게 한다.

--------------------------------
2️⃣ 하위차사 참조 탭
--------------------------------

목적

하위차사 제출 데이터를 확인하고  
필요 시 **수정요청 또는 보완요청을 보내는 화면**

레이아웃

상단

필터 영역

필터 항목

- 업체
- 차수
- 납품 항목
- 제출 상태
- 기간

--------------------------------
하위 데이터 리스트
--------------------------------

테이블 형태

컬럼 예

- 업체명
- 차수
- 납품 항목
- 제출 상태
- 최근 제출일
- 요청 상태

행 액션

- 상세 보기
- 수정요청
- 보완요청

--------------------------------
하위 데이터 상세 패널
--------------------------------

행 클릭 시 우측 패널 표시

패널 구성

섹션1

업체 정보

- 업체명
- 차수
- 납품 항목

섹션2

제출 데이터 요약

- 제출 상태
- 제출 날짜
- 데이터 버전

섹션3

요청 이력

- 수정요청 기록
- 보완요청 기록

--------------------------------
요청 액션
--------------------------------

우측 패널 하단에 버튼 제공

- 수정요청
- 보완요청

버튼 클릭 시 모달 표시

모달 내용

- 요청 유형
- 요청 메시지 입력
- 제출 버튼

--------------------------------
[중요 UX 규칙]
--------------------------------

비1차 데이터 관리 탭은 다음 원칙을 따른다.

1️⃣ 데이터 입력은 **자사 데이터 탭에서만 가능**

2️⃣ 하위차사 데이터는 **조회 및 요청만 가능**

3️⃣ 하위 데이터를 **직접 수정하는 기능은 제공하지 않는다**

4️⃣ 계산 결과는 **PCF 산정·전송 탭에서만 생성된다**

--------------------------------
[산출물]
--------------------------------

Figma에서 다음 화면을 설계한다.

- 데이터 관리 탭 전체 레이아웃
- 자사 데이터 입력 화면
- 하위차사 참조 화면
- 입력 상태 카드
- 통합 템플릿 입력 영역
- 하위 데이터 리스트
- 상세 패널
- 수정요청 / 보완요청 모달

세부 데이터 필드 정의는 하지 말고  
**화면 구조와 컴포넌트 배치 중심으로 설계한다.**
