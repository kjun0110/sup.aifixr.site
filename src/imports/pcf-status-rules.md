1. PCF 상태 계산 기준 수정

PCF 상태는 다음 규칙을 따른다.

규칙 1

어떤 회사가 하위 회사(Child Node)를 가지고 있다면
그 회사의 PCF 상태는 하위 회사 상태에 의존한다.

규칙 2

하위 회사 중 하나라도 다음 상태가 존재하면

미제출
검증중
계산 대기

상위 회사 PCF 상태는 무조건 "계산 대기"가 된다.

규칙 3

하위 회사가 모두 PCF 완료 상태일 때만

상위 회사 PCF 계산 가능

2. 상태 계산 순서

PCF 상태는 공급망 트리의 가장 하위 노드부터 계산한다.

예시

원료 공급사 C
→ 글로벌 소재
→ 한국배터리
→ 우리회사

계산 순서

원료 공급사 C
→ 글로벌 소재
→ 한국배터리
→ 우리회사
3. 현재 예시 데이터 수정

현재 UI 예시는 잘못된 상태가 포함되어 있다.

다음처럼 수정한다.

잘못된 예시
글로벌 소재 → 계산 대기
원료 공급사 C → 완료

이 상태는 허용되지 않는다.

올바른 예시
Case 1 (하위 완료)
원료 공급사 C → 완료
글로벌 소재 → 완료
한국배터리 → 완료
우리회사 → 완료
Case 2 (하위 미제출)
아시아 화학 → 미제출
글로벌 소재 → 계산 대기
한국배터리 → 계산 대기
우리회사 → 계산 대기
Case 3 (검증중)
원료 공급사 C → 검증중
글로벌 소재 → 계산 대기
한국배터리 → 계산 대기
우리회사 → 계산 대기
4. 상태 Tooltip 추가

PCF 상태 컬럼에 Tooltip을 추가한다.

내용

PCF 상태는 하위 협력사의 계산 상태에 따라 결정됩니다.
5. 트리 의존 관계 시각화

트리 아이콘 옆에 작은 연결 표시를 추가한다.

예시

우리회사
 └ 한국배터리
    └ 글로벌 소재
       └ 원료 공급사 C

목적

사용자가 PCF 계산 의존 구조를 직관적으로 이해하도록 하기 위함

핵심 수정 목표

PCF 상태는 Tier가 아니라 트리 구조(Parent → Child 관계)를 기준으로 계산되어야 한다.

그리고 하나만 말해줄게.
