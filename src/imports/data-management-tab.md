너는 제조업 공급망 ESG/PCF 관리 SaaS **AIFIX 협력사 포털**의 시니어 프로덕트 디자이너다.

현재 설계 대상은  
협력사 포털 → 프로젝트 선택 → 1차 역할 프로젝트 진입 → 네비게이션2 탭 중  
**“데이터 관리” 탭 화면**이다.

이 화면은 **1차 협력사용 화면이며 비1차 화면은 설계하지 않는다.**

--------------------------------
[고정 시스템 전제]
--------------------------------

AIFIX 협력사 포털은 다음 구조를 가진다.

- 단일 원청 전용 포털
- 협력사는 구매/PCF 직무를 분리하지 않고 **통합 템플릿 하나에 데이터 입력**
- 1차 협력사는 다음 권한을 가진다

1차 권한

- 하위차사 공급망 구조 조회
- 하위차사 데이터 조회
- 하위차사 데이터 **직접 수정 가능**
- 하위차사에 보완요청/재요청 가능
- 전체 취합 후 원청 제출

--------------------------------
[업무 흐름 규칙]
--------------------------------

협력사 업무 흐름은 반드시 다음 구조를 따른다.

1️⃣ 데이터 관리  
→ 데이터 입력 및 수정 (계산 없음)

2️⃣ PCF 산정/전송  
→ PCF 계산 실행 및 전송

3️⃣ 데이터 조회  
→ 입력 데이터 / 계산 결과 확인

따라서 **데이터 관리 화면에서는 PCF 계산 결과를 노출하지 않는다.**

--------------------------------
[화면 상단 구조]
--------------------------------

모든 네비게이션2 화면에는 **프로젝트 컨텍스트 헤더**가 존재한다.

헤더 표시 정보

- 프로젝트명
- 내 차수 (1차)
- 제출 대상 (원청사)
- 계약번호
- 마감일
- 프로젝트 상태

그 아래에 네비게이션2 탭을 유지한다.

네비게이션2

대시보드  
공급망 관리  
데이터 관리  
데이터 조회  
PCF 산정/전송  
이력

현재 화면은 **데이터 관리 탭이 활성화된 상태**이다.

--------------------------------
[데이터 관리 탭 설계 목적]
--------------------------------

데이터 관리 탭은 **PCF 계산의 재료가 되는 데이터 입력/수정 화면**이다.

이 탭에서는 다음만 수행한다.

- 자사 데이터 입력
- 하위차사 데이터 확인
- 하위차사 데이터 직접 수정
- 보완요청/재요청

PCF 계산 결과는 절대 이 화면에서 메인으로 노출하지 않는다.

--------------------------------
[데이터 관리 화면 레이아웃]
--------------------------------

화면을 다음 3개 영역으로 구성한다.

1️⃣ 상단 컨트롤 영역  
2️⃣ 데이터 입력 영역  
3️⃣ 데이터 상태/액션 영역

--------------------------------
1️⃣ 상단 컨트롤 영역
--------------------------------

상단에는 데이터 입력 상태를 관리하는 컨트롤을 배치한다.

좌측

- 데이터 버전 정보
- 마지막 저장 시각

우측 액션 버튼

- 임시 저장
- 저장
- 입력 검증

검증 버튼을 누르면  
누락 필드 / 형식 오류를 표시하는 구조로 설계한다.

--------------------------------
2️⃣ 데이터 입력 영역
--------------------------------

데이터 관리 탭 내부에는 **하위 탭 2개**를 만든다.

상단 세부 탭

자사 데이터  
하위차사 데이터

--------------------------------
A. 자사 데이터 탭
--------------------------------

목적

현재 협력사가 직접 입력하는 데이터를 관리하는 영역

레이아웃

카드 기반 섹션으로 구성한다.

예시 카드 구조

[납품 항목 정보]

- 납품 제품
- 계약 항목
- 생산 공정 구분

[구매 데이터]

- 주요 원료
- 투입량
- 공급 방식

[활동 데이터]

- 에너지 사용량
- 공정 활동 데이터

[배출계수 선택]

- 적용 배출계수
- 계수 출처

각 카드 내부에는 입력 필드 구조만 배치하고  
세부 필드 정의는 하지 않는다.

각 카드 우측 상단에는 상태 표시를 둔다.

예

- 입력 완료
- 미완성
- 검증 실패

--------------------------------
B. 하위차사 데이터 탭
--------------------------------

목적

하위차사에서 제출한 데이터를  
**1차 협력사가 조회 및 직접 수정**할 수 있는 화면

레이아웃

상단

하위차사 리스트 테이블

컬럼 예

- 업체명
- 차수
- 납품 항목
- 제출 상태
- 최근 제출일
- 보완요청 여부

리스트에서 업체를 클릭하면  
우측에 **하위차사 데이터 상세 패널**이 열린다.

--------------------------------
하위차사 상세 패널 구성
--------------------------------

섹션 1

업체 정보

- 업체명
- 차수
- 담당자
- 납품 항목
- 현재 제출 상태

섹션 2

제출 데이터 요약

- 입력된 데이터 섹션 목록
- 누락 항목 표시
- 검증 상태

섹션 3

주요 액션 버튼

1차 전용 액션

- 데이터 직접 수정
- 보완 요청 보내기
- 재요청 보내기

데이터 직접 수정 버튼을 누르면  
해당 하위차사 데이터를 편집 가능한 상태로 전환한다.

--------------------------------
3️⃣ 데이터 상태/안내 영역
--------------------------------

화면 하단 또는 우측에는  
데이터 관리 탭의 역할을 설명하는 안내 박스를 둔다.

내용 예

“이 화면에서는 PCF 계산을 위한 입력 데이터를 관리합니다.
계산은 ‘PCF 산정/전송’ 탭에서 수행됩니다.”

--------------------------------
[UX 규칙]
--------------------------------

다음 규칙을 반드시 반영한다.

1️⃣ 데이터 관리 탭은 입력/수정 전용 화면이다.

2️⃣ PCF 계산 결과(총 PCF 값 등)는 이 화면에서 메인으로 표시하지 않는다.

3️⃣ 1차 협력사는 하위차사 데이터를 직접 수정할 수 있어야 한다.

4️⃣ 모든 수정은 저장 후 버전으로 관리된다.

--------------------------------
[산출물]
--------------------------------

Figma에서 다음 화면을 생성한다.

- 네비게이션2가 포함된 “데이터 관리” 탭 화면
- 자사 데이터 / 하위차사 데이터 하위 탭 구조
- 하위차사 리스트 + 상세 패널 UI
- 저장/검증/보완요청 등의 액션 버튼

세부 데이터 필드 정의는 하지 말고  
**화면 구조와 컴포넌트 배치 중심으로 설계한다.**
